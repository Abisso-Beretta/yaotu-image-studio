# 生图任务包 · 接口规范 (v1.0)

> 这是「副导演 / 牛马制作人」(Scriptwriter) 和「妖荼」出图工具之间的**交接合同**。
> 两个工具都照这一份实现，小伙伴之间也用这个格式互传项目。
> 配套的机器可读类型见 [`shared/job-package.ts`](../shared/job-package.ts)。

---

## 1. 它在流水线里的位置

```
副导演/牛马制作人              妖荼(本工具)                     下游
(剧本 + 分镜 + 提示词)  ──→  (锁参考图 + 批量出图 + 审片 + 导出)  ──→  做视频(即梦/剪辑)
        │                          │
        └──── 生图任务包 ─────────────┘
              (一个文件夹)
```

- **上游**写文字（剧本、分镜、每镜提示词）。
- **本工具**只管把文字变成「**风格统一、角色一致**」的关键帧，然后按镜号顺序导出。
- 中间靠"生图任务包"交接。

---

## 2. 两种形态，别混淆

**(A) 导入物（Scriptwriter → 出图工具）= 单个 JSON 文件，不含图片。**
Scriptwriter 是纯文字工具，没有图，所以它只导出一个 JSON：

```
<项目>/导出/生图任务包-S01E01.json     ← 类型 = ImageJobPackage
```

**(B) 出图工具自己的项目 = 一个文件夹**（导入上面的 JSON 后，由出图工具建立并维护；它才有图片）：

```
《回声》-S01E01/              ← 出图工具的项目，可压成 zip 传给小伙伴（类型 = ImageProject）
├── project.json             ← 结构化数据（在导入的 JSON 上加：角色参考图、出图状态）
├── characters/
│   └── 林安/                 ← 注意：角色 id = 角色名
│       ├── base-01.png      ← 林安的锁定参考底图（一致性命根子）
│       └── expr-01.png
├── scenes/                  ← 可选：场景参考图
├── outputs/                 ← 出图产物
│   └── S01E01-C01-SH03/
│       ├── cand-01.png
│       └── cand-02.png
└── README.txt
```

**铁律**：图片永远是**独立文件**，绝不内联成 base64 塞进 JSON（上次 17MB 配置文件的坑就是这么来的）。JSON 只存结构和**指向文件的相对路径**。

## 3. 谁填什么

| 内容 | 谁负责 |
|---|---|
| 剧名 / 媒介 / 集号 | Scriptwriter |
| 风格锁定句 / 通用负面 | Scriptwriter |
| 角色卡（文字：外貌锚点、性格等） | Scriptwriter |
| **每镜：最终提示词 + 负面 + 出场角色** | **Scriptwriter（需补，见第 4 节）** |
| 角色**参考底图**（视觉一致性） | **本工具** |
| 出图候选、定稿、出图参数 | 本工具 |

本工具再次导出时，导出**完整包**（含上面所有），小伙伴拿到就能继续干。

## 4. ✅ Scriptwriter 侧已实现（出图工具导入时要注意的现实）

Scriptwriter 已经导出每镜的 **`prompt` + `negative` + `characterIds`**（解析自它的 `04-生图提示词`）。
对接代码在 `_软件开发/牛马制作人/src/lib/jobPackageExport.ts`、类型在 `src/types/jobPackage.ts`。

出图工具导入时必须按这些**现实**来写，别假设字段总是齐的：

- **解析是"尽力而为"**：`prompt`/`characterIds` 可能为空（04 是大模型产出的 Markdown，格式不总规整）。
  - `prompt` 为空 → 该镜在出图工具里标"待补"，让用户手填或跳过。
  - `characterIds` 漏了 → **出图工具必须支持手动给镜头指派角色**（否则挂不上参考图）。这是必备的兜底 UI。
- **`characterIds` 里是角色名**（Scriptwriter 用 `id = 角色名`），如 `["林安"]`，不是 slug。
- **`negative` 缺省**就回退 `style.globalNegative`。
- **`shotId`** 可能是完整 `S01E01-C01-SH03`，也可能降级成 `C01-SH01` / `SH01`；按它排序、去重（同号后者覆盖）。
- 除 `shotType/cameraMove/composition/scene` 外，`description/duration/...` 多为空（导出只解析了 04，没并 03 分镜表）——出图工具不要依赖它们。

```json
  },
  "defaults": { "size": "2160x3840", "model": "gpt-image-2", "attachCharacterRefs": true },
  "characters": [
    {
      "id": "林安",
      "name": "林安",
      "role": "主角",
      "appearance": {
        "imageLockPhrase": "冷白皮清瘦鹅蛋脸青年，齐肩黑色直发中分发尾微乱，狭长丹凤眼深瞳眼下淡青阴影，左眼角泪痣，黑色立领长风衣内搭深灰高领毛衣，胸前银色怀表，气质冷峻疲惫。"
      },

      ]
    }
  ],
  "shots": [
    {
      "shotId": "S01E01-C01-SH03",
      "scene": "C01",
      "shotType": "中景",
      "cameraMove": "缓慢推近",
      "composition": "居中构图，背影占画面中下，上方留招牌光",
      "duration": "4",
      "characterIds": ["林安"],
      "prompt": "中景背影，冷白皮清瘦鹅蛋脸青年，齐肩黑色直发中分发尾微乱……撑透明雨伞独自穿行霓虹巷弄……电影感赛博朋克写实风……（逐字来自 04 文件）",

```

> 上面 `characterIds: ["林安"]` → 本工具出这一镜时，会自动把 `characters/林安/base-01.png` 挂为参考图。环境空镜（SH01/SH02）`characterIds` 为 `[]`，不挂参考图。
> 注：`referenceImages` 是出图工具补的视觉层，Scriptwriter 导出的 JSON 里没有这一段。

## 6. 关键实现注意（给两边的程序）

- **`prompt` 逐字使用**：它已经包含角色外貌锚点 + 风格锁定句，本工具**不再重复拼接** style/角色文字，否则提示词会臃肿、互相打架。
- **文字一致性在上游、视觉一致性在本工具**：`prompt` 管文字，`referenceImages` 管图像，两者叠加 = gpt-image 一致性的上限。
- **负面**：优先用镜头的 `negative`，没有就回退 `style.globalNegative`。
- **尺寸**：由 `medium` 推默认（竖屏 = `2160x3840`），用户可在出图工具里覆盖；出量想省钱可选中档（见 `MEDIUM_SIZE`）。

## 7. 版本兼容

- `schemaVersion` 用 `主.次`（当前 `1.0`）。
- 读取方先看**主版本号**：主版本一致就能读（次版本只增字段、不破坏旧字段）；主版本不一致要提示用户升级工具。
